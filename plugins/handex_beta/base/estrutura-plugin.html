Plugin Figma — Arquivos principais

src/plugin/
ArquivoFunção
📋manifest.json
 configManifesto obrigatório do Figma. Define nome do plugin, versão da API, arquivo principal (code.bundle.js), UI (ui.html), editores suportados (figma, dev) e permissões — incluindo currentuser para identificar o designer logado.
🧠code.js
 fonteBackend do plugin. Roda no contexto privilegiado do Figma com acesso total ao canvas, nós, variáveis, clientStorage e API do Figma. Recebe mensagens da UI via figma.ui.postMessage, executa operações no arquivo e retorna resultados. É o arquivo editado pelos devs — não é distribuído diretamente.
📦code.bundle.js
 geradonão editarBackend compilado — gerado por npm run bundle:code. Contém code.js + audit.js em bundle único com a versão injetada. É este arquivo que o Figma executa.
🖥️ui.html
 geradonão editarFrontend completo do plugin — arquivo HTML único entregue ao Figma. Gerado por npm run bundle:ui: agrega todos os módulos JS, views HTML, estilos CSS, Tailwind, Lucide e o skeleton de referência DSC embarcado (~1.3 MB).
⚙️build.cjs
 buildScript assembler do ui.html. Lê todos os módulos JS, views HTML, CSS e dados de referência e os concatena em um único arquivo HTML. Define cores, cabeçalho, rodapé e estrutura do documento. Executado via npm run bundle:ui.
🔍audit.js
 fonteMódulo compartilhado de auditoria. Define auditProperty() (matching de tokens com score), suggestClosestMatch(), constantes AUDIT_SCORE e AUDIT_THRESHOLDS, e o template de frame JSON. Importado tanto pelo code.js (bundle) quanto pelos módulos da UI.
🧩

Módulos do Frontend

src/plugin/modules/

🧩 Cada arquivo é um fragmento JavaScript que, no build, é concatenado no <script> único do ui.html. Todos compartilham o mesmo escopo global em runtime — não há imports entre eles.

ArquivoFunção
🏛️core.js
 fonteMódulo central da UI. Inicializa o handoffData (estado global do plugin), expõe todas as funções públicas ao window, gerencia navegação entre views e steps do Handoff, persistência no clientStorage, tema (dark/light), versão, toast de notificações, accordion, scroll e utilitários gerais.
📡messages.js
 fonteDispatcher de mensagens. Implementa o único window.onmessage do plugin. Recebe todas as mensagens vindas do code.js e roteia para as funções corretas: init-plugin, scan-result, extract-done, cache-cleared, audit-cache-loaded, entre outras. Também aplica o tema do Figma na inicialização.
📤handoff.js
 fonteLógica do fluxo de Handoff. Coleta dados dos formulários (collectHandoffData), controla a navegação entre os 4 steps, exportação de dados e a função createHandoffOnCanvas() que dispara a criação da ficha técnica no canvas do Figma.
🔬specifications.js
 fonteLógica de escaneamento e exibição de specs UI. Contém scanFrame() (dispara o scan no code.js), renderSpecs(), createSpecItem(), renderActivePropsList() (com cadeia de token em breadcrumb), renderInactivePropsList() (props com valor inativo em visual diferenciado) e o sistema de toggle de props inativas.
🛡️audit.js
 fonteMódulo de auditoria da UI. Gerencia o toggle de auditoria (com modal de seleção de bibliotecas), extração de tokens via Figma Plugin API (startAuditExtraction), exibição de status de progresso, cache do bundle extraído, upload de referências manuais e o relatório final de aderência ao DSC.
📐measurement.js
 fonteMódulo de medidas e anotações. Lógica da UI para configurar e aplicar medidas visuais no canvas do Figma: gaps, paddings, larguras, alturas e dimensões de componentes selecionados.
🎨design-data.js
 fonteMódulo de dados de design. Gerencia a exibição e manipulação de dados de design system no contexto de especificações — incluindo Code Connect (mapeamento DSC → código).
🖼️

Views da Interface

src/plugin/views/

🖼️ Fragmentos HTML inseridos no ui.html durante o build. Cada arquivo corresponde a uma tela ou conjunto de componentes da interface do plugin.

ArquivoFunção
🏠home.html
 Tela inicial do plugin. Exibe as 4 ferramentas principais — Handoff, Dimensões/Medidas, Guia e Especificações — além dos botões de Exportar e Importar JSON para salvar/restaurar estado.
📝handoff.html
 Tela principal do fluxo de Handoff com 4 steps: (1) Informações & Briefing — título, status, versão, designer; (2) Specs UI — escaneamento com auditoria DSC; (3) Detalhamento — regras de negócio, docs, anexos; (4) Revisão Final — checklist de inclusão e botão Gerar Ficha no Canvas.
🔬specifications.html
 Tela de Especificações. Lista os elementos escaneados em accordions por categoria: Componentes, Ícones, Tipografia, Frames e Layouts, Vetores. Cada item mostra propriedades com cadeia de token e indicadores de auditoria DSC.
📐measurement.html
 Tela de Dimensões/Medidas. Interface para configurar tipos de medida (gap, padding, width, height) e aplicá-las como anotações visuais no canvas do Figma sobre os elementos selecionados.
📖guide.html
 Tela de Guia. Documentação de uso do plugin com instruções, fluxos e referências para os designers.
🪟modals.html
 Todas as modais compartilhadas do plugin: scan-categories-modal (seleção de categorias de escaneamento), audit-libs-modal (seleção de bibliotecas DSC ao ativar auditoria), formulários de especificação e fluxo, e demais overlays.

Referências DSC

src/plugin/refs/

⚠️ Nenhum arquivo aqui contém valores resolvidos (hex, fontSize, etc.) — apenas chaves e metadados. Os valores são resolvidos em runtime pelo Figma Plugin API. Esse design mantém o pipeline livre de tokens no cliente.

ArquivoFunção
🗂️_manifest.json
 curadoLista curada das bibliotecas DSC: slug, name, fileKey e contagens de componentes/estilos. Fonte de verdade para saber quais libs existem e como identificá-las no Figma.
🦴_skeleton.json
 geradoBundle agregado gerado por build-skeleton.cjs. Contém metadados + style keys + component keys de todas as libs (~900 KB). É embarcado em ui.html como window.__HANDEX_REF_SKELETON__ — base para a extração de tokens em runtime.
🎨fundamentos-visuais.json
 geradoReferências da lib Fundamentos Visuais DSC: keys de estilos de cor, tipografia, efeitos e keys de componentes.
🌐web-angular-react.json
 geradoReferências da lib Web Angular & React DSC.
🖥️super-gerenciador.json
 geradoReferências da lib Super Gerenciador DSC.
📱super-app.json
 geradoReferências da lib Super App DSC.
♿design-acessivel.json
 geradoReferências da lib Design Acessível DSC.
🔗code-mappings.json
 curadoMapeamento curado: nome da lib DSC → caminho de import no monorepo de código. Usado para sugestões de Code Connect — mostra ao dev qual pacote importar para cada componente identificado.
⚙️build-skeleton.cjs
 buildScript que agrega todos os {slug}.json em um único _skeleton.json. Executado via npm run bundle:refs. Sempre roda antes do build.cjs.
📡fetch-design-refs.cjs
 buildScript que busca styles e component keys via REST API do Figma. Requer FIGMA_TOKEN como variável de ambiente. Gera os arquivos {slug}.json para cada lib listada no _manifest.json. Executado via npm run refs:fetch.
📄README.md
 Documentação do pipeline de referências: comandos disponíveis, estrutura de arquivos, setup do CI/CD no GitLab, gestão segura do FIGMA_TOKEN e instruções de rotação de credenciais.

⚡ Fluxo de build completo

1
npm run refs:fetch → src/plugin/refs/{slug}.json
Busca styles e component keys nas libs DSC via REST API do Figma. Requer FIGMA_TOKEN. Rodado apenas quando o DSC é atualizado.
2
npm run bundle:refs → src/plugin/refs/_skeleton.json
Agrega todos os {slug}.json em um único skeleton compacto que será embarcado na UI.
3
npm run bundle:ui → src/plugin/ui.html
Roda bundle:refs + build.cjs. Monta o HTML completo do frontend: views, módulos JS, CSS, Tailwind, Lucide e skeleton embarcado.
4
npm run bundle:code → src/plugin/code.bundle.js
Compila code.js + audit.js com esbuild, injetando __HANDEX_VERSION__. Gera o backend final que o Figma executa.
Artefatos finais entregues ao Figma: ui.html (frontend) + code.bundle.js (backend) — ambos gerados, nunca editados diretamente.

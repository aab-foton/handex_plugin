Como usar o plugin

O Handex organiza toda a documentação de handoff em ferramentas independentes. Use cada uma no momento certo — não precisa seguir uma ordem. Quando tudo estiver pronto, gere a Ficha de Handoff no canvas.

  1. 1Clique em Informações do Projeto na home — o modal abre sobre qualquer tela, sem precisar sair do que está fazendo.
  2. 2Preencha Título, Versão e Objetivo — esses três campos são obrigatórios para gerar a ficha.
  3. 3Adicione ao menos 1 membro de equipe com e-mail válido e papel Designer — sem isso o botão de gerar ficha fica bloqueado.
  4. 4Opcional: ative o Briefing Estratégico para responder perguntas de contexto (escopo, stakeholders, UX), adicione Regras de Negócio e cole links de protótipo, acessibilidade e pesquisa.
  5. 5Tudo é salvo automaticamente a cada alteração — não há botão de salvar.
  1. 1Selecione um Frame (ou Componente, Seção, Grupo) diretamente no canvas do Figma antes de abrir esta ferramenta.
  2. 2Clique em + Registrar Frame — o plugin captura o ID e o nome do nó selecionado e cria um card de frame.
  3. 3Dentro do card, escolha as categorias de scan (Componentes, Ícones, Tipografia, Frames/Layouts, Vetores) e confirme.
  4. 4O resultado aparece agrupado por tipo. Clique em qualquer item para destacar o elemento no canvas.
  5. 5Declare a Conformidade DSC: marque "Check Designs realizado", informe se há desvios e adicione observações. Desvios são salvos como ressalvas e aparecem na ficha.
  6. 6Se o frame é um Novo Componente, ative o toggle — a seção de conformidade é ocultada (não se aplica a componentes inéditos) e o frame é destacado na ficha.
  7. 7Repita para cada frame do projeto. Você pode ter múltiplos frames documentados em paralelo.
  1. 1Selecione um elemento no canvas do Figma — pode ser um componente, texto, ícone ou qualquer nó.
  2. 2Clique no botão + no rodapé da view de Especificações. O formulário abre com o nó selecionado já vinculado.
  3. 3Defina a Letra (tag de referência, ex: A, B, C1), a Categoria (Comportamento, Regra, Layout, API etc.) e a Cor do grupo.
  4. 4Adicione uma Nota técnica e, se aplicável, um link para o componente no DSC.
  5. 5Em Propriedades, adicione os atributos técnicos da spec: nome da propriedade, token do Design System e valor aplicado.
  6. 6A spec é criada à direita do elemento no canvas, com a letra visível. Mesma letra empilha verticalmente; letra diferente abre nova coluna.
  7. 7Na lista, você pode nomear grupos de letra (ex: "Grupo A — Botão Primário") e ocultar grupos inteiros da ficha.
  8. 8Para documentar cenários alternativos, expanda uma spec e clique em + Exceção — escolha Erro, Sucesso, Alerta ou Confirmação.

Exportar

  1. →Acesse Gerar Ficha de Handoff → use os botões de exportação no rodapé para baixar JSON (backup completo), Markdown ou HTML interativo.
  2. →O JSON é a fonte completa de dados — contém specs, medidas, fluxos, propriedades e tokens escaneados. Ideal para integrações com outros sistemas.

Importar

  1. →Clique em Importar JSON na home e selecione um arquivo exportado anteriormente. O plugin restaura todos os dados e incrementa a versão automaticamente.
  2. →Após importar, escolha o que recriar no canvas: Ficha Técnica, Cards de Spec e/ou Medidas.

Documentação de Apoio

Layers geradas pelo plugin são bloqueadas

Specs, medidas, fluxos e a Ficha de Projeto são criados com bloqueio de edição para preservar a integridade do handoff. Para remover ou recriar um item, utilize os botões de exclusão dentro do próprio plugin.

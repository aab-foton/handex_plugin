Tipo de especificação

Identificadas por tags, servem para a rápida identificação visual do tipo de informação contidas nos indicadores e notas de especificação. Confira a seguir:

Cenário de exceção

Descrição: Estados alternativos ou finais (Sucesso, Erro, Alerta). Mensagens de erro de API ou alertas de sistema.

Exemplo: Sucesso, Erro, Alerta, Falha de validação, Fluxos não ideais.

Informação extra

Descrição: Informações essenciais que não se encaixam em outras categorias. Pendências, contextos adicionais ou decisões de reunião.

Exemplo: Alertas, Observações, Pendências, Contexto adicional, Decisões futuras, Componente legado ou fora do DSC.

Comportamento

Descrição: Reação do sistema às ações do usuário além do padrão do DSC. Microinterações, abertura de modais e transições de estado.

Exemplo: Micro interações, Sequência de ações, Comportamento contextual da jornada, Comportamento específico do contexto.

Regras de Negócio

Descrição: Lógica do sistema não visível apenas pela interface. Campos obrigatórios, validações e restrições de ações.

Exemplo: Campos obrigatórios, Combinações válidas/inválidas, Limitações de ações, Processos condicionais.

Dados de API

Descrição: Informações técnicas para integração com o backend. Endpoints, campos esperados e estados de carregamento.

Exemplo: Telas dependentes de resposta de API, Dados ainda inexistentes, Informações temporárias ou mockadas.

Layout

Descrição: Descrição de grids, margens, alinhamentos e estruturas de containers.

Exemplo: Grids, Espaçamentos, Breakpoints, Max-width.

Interação

Descrição: Descrição de fluxos de navegação e gestos do usuário.

Exemplo: Swipes, Taps, Navegação entre telas.

Acessibilidade

Descrição: Notas sobre contraste, áreas de toque e leitura por leitores de tela.

Exemplo: Alt-text, Hierarquia de cabeçalhos, Focus states.

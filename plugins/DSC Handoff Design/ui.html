Instruções de uso

  1. O plugin aceita Component Set (com variantes) ou Component avulso. Também é possível selecionar uma instância ou variante — o plugin resolve automaticamente.
  2. Não renomeie o handoff gerado. O nome segue o padrão [dsc] Handoff: [Nome do componente] e é usado pelo plugin para localizar e atualizar o documento.
  3. Siga o padrão de nomenclatura nas propriedades do componente:
    • Visibilidade: prefixe com show (ex: show Icon)
    • Texto: use label ou text (ex: label Button)
    • Variações visuais: use Variant (ex: cor, estilo, tamanho)
    • Estados: use State (ex: default, hover, focused)
  4. A tabela de propriedades segue a mesma ordem exibida no painel do Figma. Reorganize as propriedades no componente para controlar a ordem no handoff.
  5. A maioria das descrições de propriedades já está mapeada. Quando não houver uma descrição, o plugin sinalizará com um alerta para preenchimento manual.
  6. O dark mode nas variações só é gerado quando o componente não possui propriedade State. Se o componente tiver State, o dark mode aparece na matriz de estados.
  7. Ao atualizar com "Manter alterações manuais" ativado, textos editados e posições de badges na anatomia são preservados. Se o número de elementos do componente mudar, as posições são recalculadas automaticamente.
  8. O plugin pode cometer erros. Revise sempre o conteúdo gerado antes de publicar.

Instruções de Uso

Plugin para gerar a documentação de acessibilidade de componentes do Design System Caixa.

Como começar

  1. Certifique-se de que há uma instância do template [dsc-h] Template Handoff na página
  2. Selecione o template no canvas — o plugin exibirá “Template carregado”
  3. Selecione qualquer layer dentro do componente que deseja anotar — o plugin abre as abas de edição

A partir daí você pode navegar livremente pelo canvas. O plugin acompanha a seleção ativa e atualiza os dados automaticamente.

Tipos de seleção aceitos para o componente:

  • Qualquer layer dentro do componente — o plugin resolve automaticamente para o component set pai
  • Instância — inicia na variante da instância selecionada
  • Handoff gerado — resolve para o componente de origem (permite editar e atualizar)

Todos os dados são salvos no component set original, independente do tipo de seleção.

Importante: não renomeie a instância do template nem o handoff gerado. O plugin identifica o template pelo nome [dsc-h] Template Handoff e o handoff pelo prefixo [A11Y Handoff].

Componentes de library (outro arquivo):

O plugin consegue ler dados de componentes de library, mas não salvar — componentes de outro arquivo são somente leitura. Para editar, abra o plugin diretamente no arquivo original da library.

Aba Geral

  • Plataformas: selecione as plataformas alvo (Web, Mobile, etc.)
  • Zoom: marque os tipos de zoom suportados (reflow, resize text)
  • Mapeamento de teclado e gestos: selecione quais teclas e gestos o componente suporta. Os dados vêm do CSV carregado nas Configurações. Use o botão ✎ em um item para removê-lo e abrir o campo de busca pré-preenchido para facilitar a troca

Configurações (CSV)

Acesse pelo ícone de engrenagem no topo do plugin.

  • CSV de Teclado e Gestos: colunas Mapeamento, Descrição, Utilização
  • CSV de Roles: colunas Role, Specification, Description, Observation, Nota de Código Web, Nota de Código React Native, Accessible Name, Connector Type, Revision
  • Ao carregar o CSV, os dados são salvos automaticamente no arquivo Figma e no seu perfil local. Outros usuários que abrirem o mesmo arquivo já terão os dados disponíveis sem precisar carregar novamente
  • Para atualizar, basta carregar o CSV novamente — a versão mais recente (por timestamp) será usada automaticamente
  • Ao abrir outro arquivo que já tenha o CSV salvo, os dados são sincronizados automaticamente para o seu perfil

Aba Área de Toque

  1. Clique em “Adicionar nova área de toque”
  2. Escolha o tamanho: Aprimorado (44px), Mínimo (24px) ou Exato (usa as dimensões reais do layer)
  3. Selecione o elemento diretamente no canvas do Figma — a área é posicionada automaticamente sobre o layer
  4. Repita para adicionar mais áreas na mesma sessão
  5. Clique em “Concluir” para salvar ou “Cancelar” para descartar

Modos de tamanho:

  • Aprimorado (44px) e Mínimo (24px) centralizam a área sobre o layer com o tamanho fixo escolhido. A forma pode ser Largura total (usa a largura do layer) ou Quadrado
  • Exato usa os bounds reais do layer — ideal quando o alvo de toque já tem o tamanho correto no Figma

Editar uma área existente:

  • Clique no botão ✎ ao lado da área na lista para abrir os campos X, Y, Largura e Altura
  • Os valores são aplicados em tempo real enquanto você digita

As áreas são validadas segundo WCAG 2.5.8 (mínimo 24×24px, aprimorado 44×44px).

Aba Tabulação

  1. Clique em “Adicionar ordem de foco”
  2. Selecione um elemento diretamente no canvas do Figma — ele aparece como item pendente no plugin
  3. Clique em “Adicionar à ordem” para confirmá-lo com o próximo número sequencial
  4. Repita para cada elemento na ordem de tabulação desejada
  5. Clique em “Concluir” para salvar ou “Cancelar” para descartar

Reordenar itens:

  • Use os botões ↑ e ↓ ao lado de cada item para ajustar a posição sem precisar refazer
  • Use o × para remover um item individualmente

Marque “Este componente não tem tabulação” se o componente não participa da ordem de foco.

Aba Leitor de Tela

As anotações são vinculadas a layers selecionados diretamente no canvas do Figma.

Anotação Simples ou Combinada:

  1. Selecione o layer desejado no canvas do Figma — o nome aparece indicado no topo da aba
  2. Clique em “Adicionar anotação” — o formulário abre já vinculado ao layer
  3. Escolha o modo Simples ou Combinado e preencha Tipo de Conector, Role e Especificação
  4. Descrição, Observações, Nome Acessível e Notas de Código são preenchidos automaticamente do CSV
  5. Clique em “Salvar”

Também é possível abrir o formulário primeiro e selecionar o layer depois — o campo de layer atualiza em tempo real.

Agrupamento:

  1. Selecione dois ou mais layers no canvas (Shift+clique) — o plugin detecta a multi-seleção
  2. Clique em “Adicionar anotação” e escolha o modo Agrupamento
  3. Os bounds do retângulo são preenchidos automaticamente a partir da seleção
  4. Selecione Tipo de Agrupamento, Role e Especificação e clique em “Salvar”

Editar uma anotação existente:

  • Clique no item na lista ou use o botão ✎ ao lado dele
  • O formulário abre pré-preenchido — ajuste e clique em “Salvar”

Notas de código por plataforma:

  • O CSV de Roles possui colunas separadas para Web e React Native
  • O handoff exibe automaticamente a nota da plataforma selecionada na aba Geral
  • Com Web + Mobile Cross-platform e notas diferentes, ambas são exibidas com label
  • Plataformas Android e iOS nativos não exibem nota de código por enquanto

Aba Handoff

  • Gerar Handoff: cria o documento de acessibilidade completo ao lado do componente
  • Atualizar Handoff: atualiza um handoff existente com os dados atuais do plugin

O que é atualizado:

  • Todas as seções são regeradas com os dados atuais (mapeamentos, áreas de toque, tabulação, leitor de tela, zoom)
  • Previews, conectores e posicionamento de marcadores são recalculados
  • Dimensões de áreas de toque e ordem de foco refletem os dados do plugin

O que pode ser preservado:

  • Com “Manter alterações manuais” ativado (Configurações), textos editados diretamente no handoff (descrições, observações, notas) são preservados se foram modificados manualmente
  • O plugin compara com o baseline da geração anterior — só preserva textos que você editou, não valores auto-gerados que mudaram

Atualização seletiva:

  • Ao atualizar, você pode escolher quais seções regenerar usando os checkboxes
  • Seções desmarcadas são preservadas integralmente — posições, tamanhos, textos e qualquer edição manual permanecem intactos
  • Seções marcadas são destruídas e recriadas com os dados atuais do plugin

Configurações (engrenagem):

  • Sincronizar template: atualiza a estrutura visual do handoff com o template original (recomendado manter ativado)
  • Manter alterações manuais: preserva textos editados diretamente no Figma nas seções que forem regeneradas

Preservação automática ao regenerar:

  • Posições de marcadores: conectores, badges e marcadores reposicionados manualmente no preview são restaurados após a regeneração (matching por nome + texto)
  • Textos manuais: textos editados diretamente no handoff são preservados via comparação com baseline

Limitação: mudanças de estilo visual (cores, fontes aplicadas manualmente) em seções regeneradas não são preservadas. Para manter edições de estilo, desmarque a seção antes de atualizar.

Variantes e Exceções

  • Se o componente é um Component Set com variantes, os dados são compartilhados por padrão
  • Marque “Alguma variante tem X diferente?” para criar exceções por variante
  • Selecione a variante na lista para editar seus dados específicos
